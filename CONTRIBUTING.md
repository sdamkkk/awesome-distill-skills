# Contributing to Awesome Distill Skills

Thank you for your interest in contributing! This document provides guidelines for submitting new projects and maintaining the quality of this curated list.

## 📋 Inclusion Criteria

A project must meet **all** of the following to be listed:

1. **Open Source**: The project must be publicly available on GitHub (or similar platform) with accessible source code.
2. **Actual Output**: The project must produce a usable Skill, Prompt, or Agent — not just a concept or blog post.
3. **Methodology Labeled**: The distillation methodology must be identifiable (prompt engineering, cognitive framework, chat-based, etc.).
4. **License Specified**: The project's license must be clearly stated. If no license is found, it will be marked as "License N/A".
5. **Ethical Standards**: The project must not promote harassment, stalking, or privacy invasion.

## 🚀 How to Submit a New Project

### Option 1: Pull Request (Preferred)

1. Fork this repository
2. Add your project entry to the appropriate category in both `README.md` and `README_CN.md`
3. Follow the entry format below
4. Submit a pull request

### Option 2: Open an Issue

If you're not comfortable with PRs, [open an issue](../../issues/new) with the project details.

## 📝 Entry Format

### English (README.md)

```markdown
- **[author/project-name](https://github.com/author/project-name)** - One-line description. `methodology-tag` `language` `license`
  - Distilled persona: Who was distilled
  - Highlights: One sentence about what makes it special
```

### Chinese (README_CN.md)

```markdown
- **[author/project-name（中文名）](https://github.com/author/project-name)** - 一句话描述。`方法论标签` `语言` `License`
  - 蒸馏对象：XXX
  - 亮点：一句话说明特色
```

### Methodology Tags

Use one of these standardized tags:

| Tag | Description |
|---|---|
| `cognitive-framework` | Extracts mental models and decision heuristics from public materials |
| `chat-based` | Creates persona from private chat logs and messages |
| `universal-framework` | General framework for distilling any person |
| `methodology-extraction` | Focuses on extracting methodologies, not personas |
| `multi-persona-debate` | Multiple personas debating the same question |
| `prompt-engineering` | Pure prompt-based persona creation |
| `bazi-astrology` | Based on Chinese astrology (Bazi) |
| `auto-research` | Automated research and distillation pipeline |

### Category Selection

Place your project in the most fitting category:

| Category | For projects that... |
|---|---|
| 🧠 Cognitive Framework | Extract deep mental models from public materials |
| 💬 Chat-based | Use private chat logs as primary source |
| 🔧 Universal Frameworks | Provide general-purpose distillation tools |
| 👨‍💼 Business Leaders | Focus on entrepreneurs and business figures |
| 🎓 Educators & Mentors | Focus on teachers, advisors, mentors |
| 📚 Thinkers & Authors | Focus on philosophers, writers, thought leaders |
| 🎭 Other Personas | Everything else (fictional, astrology, etc.) |
| 📋 Awesome Lists | Other curated lists in this space |

## ⚖️ License Requirements

- **Must specify**: Every submitted project entry must include the project's license
- **Accepted licenses**: MIT, Apache 2.0, GPL, CC, or any OSI-approved license
- **No license found**: Mark as `License N/A` — we'll still list it but note the absence
- **Proprietary**: Not accepted for this list

## 🔍 Quality Standards

We aim for quality over quantity. Before submitting, please check:

- [ ] The project has a meaningful README
- [ ] The distillation actually works (not just a placeholder)
- [ ] The methodology is described or can be inferred
- [ ] The project is maintained (not abandoned with critical issues)
- [ ] No duplicate entries exist in the list

## 🚫 What We Don't Accept

- Projects that only exist as blog posts without code/skills
- Projects that require paid access with no open-source component
- Projects promoting harassment, stalking, or non-consensual surveillance
- Projects that violate the privacy of real individuals without consent
- Duplicate entries of the same project
- Self-promotional spam without substance

## 📐 PR Template

When submitting a PR, please include:

```
## New Project Submission

**Project**: [author/project-name](URL)
**Category**: (e.g., 🧠 Cognitive Framework)
**Methodology**: (brief description)
**License**: (e.g., MIT)
**Distilled Persona**: (who is distilled)
**Why it should be listed**: (1-2 sentences)

### Checklist
- [ ] Project is open source
- [ ] Project produces a usable Skill/Prompt/Agent
- [ ] License is specified
- [ ] Entry added to both README.md and README_CN.md
- [ ] Entry follows the standard format
- [ ] No duplicate entries
```

## 🔄 Updating Existing Entries

If you notice outdated information (wrong star count, changed license, renamed project), feel free to submit a correction PR. No template needed — just describe the change.

## 💬 Questions?

Open an issue with the `question` label and we'll get back to you.
